# About

I'm an IT support professional with 4+ years of experience, currently working as a Senior Helpdesk Analyst / L2 Support. I hold ITIL Foundation, Microsoft AZ-900 (Azure Fundamentals), and Microsoft DP-900 (Data Fundamentals) certifications, and I'm currently studying toward AZ-104 (Azure Administrator).

I'm moving toward infrastructure and cloud-focused roles - Cloud Support Engineer, Junior Cloud Engineer, Infrastructure Engineer, or similar. This project is how I've been building that experience directly rather than only through study material: by running a real environment, breaking it in real ways, and fixing it properly.

## Why this project exists

Reading about DNS resolution order or Docker networking is one thing. Being paged (by my own ntfy notifications) at midnight because Pi-hole's IPv6 upstream silently broke resolution for every device on the network, and having to actually trace it from symptom to root cause under time pressure, is a different kind of understanding. That gap is what this project is meant to close.

Everything on this site reflects something that actually happened while operating this environment - the working configuration, but also the incidents, the wrong turns, and the fixes. I've kept the debugging process visible rather than only publishing the clean end state, because the process is closer to what the job actually looks like.

Roles where I can keep doing this kind of work with more scale and more consequence: L2/L3 support, infrastructure engineering, MSP environments, or junior cloud/DevOps positions where hands-on troubleshooting and systems thinking matter more than any single technology on a resume.
